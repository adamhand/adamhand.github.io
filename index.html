maven repository 配置

发表于 2020-02-26 | 分类于 工具

问题描述

开发时遇到了这样一个问题：存在两个项：admin和thrit，其中thrit中发布的jar包需要在admin端进行调用，两个项目都是使用maven管理的。但是在thrit中新建类之后，使用mvn compile package install命令将jar包部署到本地仓库，在admin端却import不进来。

最后发现maven安装jar包的仓库地址和import的地址是不同的。

通过查看maven安装jar包的记录如下：

jar包安装到了D盘。

但是在idea的File | Settings | Build, Execution, Deployment | Build Tools | Maven中查看的Local Repository路径却是另一个，在C盘，导致安装的jar包找不到。原来是因为使用了两个setting.xml的原因。

maven有2个位置可以选择配置M2_HOME/conf/setting.xml和~/.m2/setting.xml，前者位于安装目录，是全局性的，后者只针对用户的。我在M2_HOME/conf/setting.xml配置了<local repository>的路径为D:盘，在~/.m2/setting.xml中没有配置，所以jar包会安装在D盘，但是在idea中配置的Local Repository路径却在C盘，这就导致了找不到安装的jar包。

如果在~/.m2/setting.xml也配置了repository地址，会将M2_HOME/conf/setting.xml中的配置覆盖带掉。

总结就是，在idea中配置的仓库地址和在配置文件中配置的地址一定要相同。

谷歌浏览器开发者模式的各种功能简单小结。

开发者工具包括 Elements 、 Console 、 Sources 、 Network 等多个标签页，分别提供了以下功能：

  • Elements：显示网页经过渲染之后的结构，可以任意调整和修改网页元素，并即时显示修改结果；
  • Console：打印变量信息，用于代码调试，网页运行过程中产生的警告和报错也会出现在这里；
  • Sources：查看网页所使用到的全部资源文件；
  • Network：查看网页所请求的各类资源文件及其对应的请求时间。

Network 标签页会记录网页在渲染过程中所请求的各类资源文件及其对应的请求时间。大多数网页只在一开始加载的时候请求各类资源文件，加载完毕后不再请求；也有一些网页在加载完毕后仍定时请求一些资源，用于动态更新页面上的内容。所访问的网页使用了哪些资源，用户浏览的过程中网页做了哪些事情，这些都可以在 Network 标签页中找到答案。

Network 标签页中的资源文件主要分为以下几大类：

  • All：不加筛选条件，所请求的全部资源文件；
  • XHR：异步请求的数据；
  • JS：js代码文件；
  • CSS：css样式文件；
  • Img：jpg、png等图片文件；
  • Media：媒体资源文件；
  • Font：字体文件；
  • Doc：静态html文档。

idea和pycharm中windows编辑文件放到linux的换行符问题

发表于 2019-05-05 | 分类于 杂

windows中的换行符和Linux的换行符不同，有可能导致在windows下编写的程序上传到linux上时出错。几种常用系统的换行符如下：

  • windows: \r\n
  • linux\unix\macOS: \n
  • 早期的macOS: \r

有两种方法解决这种换行符的问题。

第一种，将程序上传到linux之后，使用vim打开文件，执行set ff=unix，这时，就将文件中的windows换行符变成了linux换行符。但是这种方法比麻烦，如果有很多的文本文件，需要一个一个改。

第二种，使用idea和pycharm自带的功能。首先选中需要改变换行符的项目的根目录，然后点击File - Line Separators，接着就可以选择换行符了，一共有三种可以选：windows、unix and macOS和Classic macOS。
