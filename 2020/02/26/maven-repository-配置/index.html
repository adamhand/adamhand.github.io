maven repository 配置

发表于 2020-02-26 | 分类于 工具

问题描述

开发时遇到了这样一个问题：存在两个项：admin和server，其中admin中发布的jar包需要在server端进行调用，两个项目都是使用maven管理的。但是在admin中新建类之后，使用mvn compile package install命令将jar包部署到本地仓库，在server端却import不进来。

最后发现maven安装jar包的仓库地址和import的地址是不同的。

通过查看maven安装jar包的记录如下：

但是在idea的File | Settings | Build, Execution, Deployment | Build Tools | Maven中查看的Local Repository路径却是另一个，导致安装的jar包找不到。原来是因为使用了两个setting.xml的原因。

maven有2个位置可以选择配置M2_HOME/conf/setting.xml和~/.m2/setting.xml，前者位于安装目录，是全局性的，后者只针对用户的。我在M2_HOME/conf/setting.xml配置了<local repository>的路径为D:盘，在~/.m2/setting.xml中没有配置，所以jar包会安装在D盘，但是在idea中配置的Local Repository路径却在C盘，这就导致了找不到安装的jar包。

如果在~/.m2/setting.xml也配置了repository地址，会将M2_HOME/conf/setting.xml中的配置覆盖带掉。

总结就是，在idea中配置的仓库地址和在配置文件中配置的地址一定要相同。
